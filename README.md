## Summary

Open source accounting system for Kenyan biz. Payroll first, more l8r.

## Setup

```bash
pip install -r requirements.txt
```

### Google Sheets integration

The app uploads updated leave stocks to a Google Sheets spreadsheet (`leave_stocks_YYYY`). This requires OAuth credentials:

1. Place your Google Cloud OAuth client credentials (installed app type) at:
   ```
   ~/.config/google/everyday_creds.json
   ```
2. On first use, gspread will open a browser for authorization and cache the token at:
   ```
   ~/.config/google/gspread_authorized_user.json
   ```
   Required scopes: `spreadsheets` and `drive`.

The target spreadsheet (e.g. `leave_stocks_2026`) must already exist and be accessible to the authorized Google account.

## Private Data Directory

KenyAccounting expects a sibling directory at `../el/payroll/` containing private employee data. This directory is **not** part of this repository and must be set up separately.

### Expected layout

```
../el/payroll/
├── inputs/                          # All payroll input files
│   ├── IDs-Certs - IDs.tsv          # Employee IDs, KRA PINs, NSSF/SHIF numbers
│   ├── master_employees.tsv         # Consolidated employee registry
│   ├── contracts.tsv                # Contract terms & effective salaries
│   ├── Salaries - *.tsv             # Monthly salary calculation spreadsheets
│   ├── StaffCalendarAttendance_2025 - Daily.tsv   # Daily attendance pivot table
│   ├── leave_stocks/                # Monthly leave balance snapshots
│   │   └── YYYY/
│   │       └── leave_stocks_YYYY_MM_DD.tsv   # Balances as of end of month
│   ├── timesheets/                  # Normalized timesheets
│   │   └── YYYY_MM.tsv             # Per month
│   ├── ocr_contracts/               # OCR'd text extracted from signed contracts
│   ├── SignedCgContracts/           # Scanned caregiver contract PDFs
│   └── SignedMgrOfcContracts/       # Scanned manager/office contract PDFs
├── outputs/                         # Generated payroll outputs (per month)
│   └── YYYY_MM/
│       ├── payslips/                # Individual payslip text files
│       ├── bank_payment.csv         # Equity Bank bulk salary CSV
│       ├── kra_p10.csv              # KRA P10 PAYE return
│       ├── nssf_return.csv          # NSSF statutory return
│       ├── sha_return.csv           # SHA/SHIF statutory return
│       ├── leave_stocks_YYYY_MM_DD.tsv  # Leave balances after this period
│       └── summary_table.tsv       # Payroll summary
└── tools/
    └── convert_attendance.py        # Attendance pivot → normalized timesheet
```

### Data flow

1. **inputs/** contains all payroll input files:
   - `IDs-Certs - IDs.tsv` is the gold standard for identity data
   - `master_employees.tsv` is populated from IDs-Certs
   - `contracts.tsv` is populated from OCR'd contracts + salary spreadsheets
   - `timesheets/YYYY_MM.tsv` from attendance pivot via `convert_attendance.py`
2. **outputs/** are generated by running payroll (`run_payroll.py` or the Streamlit app)
3. **leave_stocks/** are updated after each payroll run (saved locally and uploaded to Google Sheets)

### Leave stocks

Each payroll month reads leave balances from the prior month's file. For example, February 2026 payroll reads `leave_stocks_2026_01_31.tsv`. After running payroll, the updated balances are saved as `leave_stocks_2026_02_28.tsv` and can be uploaded to the `leave_stocks_2026` Google Sheet as tab `2026_02_28`.

### Converting attendance to timesheets

```bash
cd ../el && python payroll/tools/convert_attendance.py --year 2026 --month 1
```

### Running payroll

```bash
python run_payroll.py --year 2026 --month 1       # generates outputs
python run_payroll.py --year 2026 --month 1 --no-save  # preview only
```

Or use the Streamlit app: `streamlit run app.py`
